Designed for:

- Docker
- Git
- Easy deployment
- Easy backup
- Easy migration
- Easy maintenance

# Folder Description

## docker/

Contains everything related to Docker.

- Dockerfile
- Docker Compose
- Entrypoint
- Master/Caves startup scripts

Docker never stores persistent game data.

---

## config/

Contains the default server template.

Examples:

- cluster.ini
- server.ini
- worldgenoverride.lua
- modoverrides.lua

This folder is tracked by Git.

Never store save games here.

---

## data/

Runtime data generated by the server.

### data/cluster/

Contains

- save games
- player data
- generated world

This is the most important folder.

### data/server/

Contains

- DST Dedicated Server
- SteamCMD installation
- downloaded workshop mods

This folder can be recreated if necessary.

### data/logs/

Contains runtime logs.

Examples

- master.log
- caves.log
- update.log
- backup.log

---

## mods/

Contains mod configuration.

```
mods.txt
```

Example

```
378160973
3625456699
```

The installer reads this file automatically.

---

## secrets/

Contains private files.

Examples

```
cluster_token.txt
```

Never commit secrets.

Only commit

```
cluster_token.txt.example
```

---

## backups/

Stores backup archives.

Example

```
dst-2026-07-02.tar.gz
```

---

## scripts/

Automation scripts.

```
install.sh
```

Install dependencies.

```
install-mods.sh
```

Download Workshop mods.

```
update.sh
```

Update DST Dedicated Server.

```
backup.sh
```

Backup runtime data.

```
restore.sh
```

Restore backup.

```
start.sh
```

Start Docker containers.

```
stop.sh
```

Stop Docker containers.

```
restart.sh
```

Restart Docker containers.

```
logs.sh
```

View Docker logs.

```
status.sh
```

Show Docker status.

---

# Backup Strategy

Backup only

```
data/
```

Especially

```
data/cluster/
```

Everything else can be recreated from Git.

---

# Mod Management

Only edit

```
mods/mods.txt
```

Never edit scripts when adding a new mod.

---

# Secret Management

Never upload

```
cluster_token.txt
```

Always upload

```
cluster_token.txt.example
```

---

# Project Philosophy

GitHub stores:

- Source code
- Docker
- Scripts
- Configuration

Docker runs:

- DST Dedicated Server

Runtime stores:

- Save game
- Logs
- Mods
- World

Backups protect:

- Runtime data
